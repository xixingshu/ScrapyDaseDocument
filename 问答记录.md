## Q1: 介绍 scrapy 的 start_requests 和 scrapy_redis 的 start_requests

**A:**

### Scrapy 的 `start_requests`

在普通 Scrapy 中，`start_requests()` 是 Spider 的入口方法。默认实现根据 `start_urls` 列表生成初始 Request：

```python
def start_requests(self):
    for url in self.start_urls:
        yield scrapy.Request(url=url, callback=self.parse)
```

- 爬虫启动时执行一次
- 从 `start_urls` 读取 URL，逐个 yield Request
- **请求队列在内存中**，爬虫停止后丢失
- 不支持分布式（多个爬虫实例会重复爬取）

### scrapy-redis 的 `start_requests`

scrapy-redis 的 Spider 会**重写** `start_requests()`，改为从 Redis 队列中读取 URL，而非从 `start_urls`：

```python
def start_requests(self):
    yield from self.scheduler.next_requests()
```

- **Redis 作为共享请求队列**，多个爬虫实例消费同一个队列 -> 天然分布式
- **断点续爬**：爬虫停止后，Redis 中的请求不会丢，重启后继续消费
- **去重**：使用 Redis 的集合（set）做请求指纹去重，避免重复爬取

使用方式：

```python
from scrapy_redis.spiders import RedisSpider

class MySpider(RedisSpider):
    name = "myspider"
    redis_key = "myspider:start_urls"

    def parse(self, response):
        # ...
```

启动后需要用 redis-cli 推入起始 URL：

```bash
redis-cli lpush myspider:start_urls "https://example.com"
```

### 对比

| 特性 | Scrapy | scrapy-redis |
|------|--------|-------------|
| 请求队列 | 内存 | Redis |
| 分布式 | 不支持 | 支持 |
| 断点续爬 | 不支持 | 支持 |
| 去重 | 内存 set | Redis set |
| 启动方式 | 自带 `start_urls` | 手动向 Redis 推入 URL |

## Q2: isinstance 是干什么的？

**A:** `isinstance(obj, classinfo)` 用于检查一个对象是否是指定类（或类型）的实例，返回 `True` 或 `False`。

```python
isinstance(123, int)             # True
isinstance("abc", str)           # True
isinstance([1,2], (list, tuple)) # True（支持多个类型）
isinstance("abc", int)           # False
```

第二个参数也可以是类型元组，只要匹配其中之一就返回 `True`。常用于类型判断和参数校验。

## Q4: 为什么 scrapy 里的 start_requests 弃用，新版和旧版区别在哪里？

**A:** Scrapy 2.13+ 引入 `async def start()` 替代 `start_requests()`，目的是统一 Spider 入口的异步化。

**弃用原因：**
1. 让 Spider 入口支持 `async`，与 Scrapy 内部架构对齐（回调、中间件已全部 async）
2. 允许 `start()` 直接 yield Item（dict），旧版只能 yield Request

### 新旧对比

| 对比 | 旧版 `start_requests()` | 新版 `async start()` |
|---|---|---|
| 类型 | 同步生成器 `def` | 异步生成器 `async def` |
| 返回值 | 只能 yield `Request` | 可 yield `Request` **或** Item（dict） |
| 默认实现 | 直接遍历 `start_urls` | 默认委托给 `start_requests()` 做兼容 |

### 新版 `start()` 示例

```python
# yield Item（旧版做不到）
async def start(self):
    yield {"foo": "bar"}

# async 原生支持协程
async def start(self):
    data = await fetch_something_async()
    yield Request(data["url"])
```

### 新版默认实现（与旧版完全等价）

```python
async def start(self):
    with warnings.catch_warnings():
        warnings.filterwarnings("ignore", category=ScrapyDeprecationWarning, ...)
        for item_or_request in self.start_requests():
            yield item_or_request
```

实际功能完全一致，只是包了一层兼容层。对大多数用户无影响——继续用 `start_requests` 或直接覆盖 `start()` 都行。

## Q5: 如果我想通过 scrapy_redis 发送 get、put、delete 请求的话该如何重写？

**A:** scrapy-redis 默认的 `make_request_from_data` 统一使用 `FormRequest`，但 `FormRequest` 的 `formdata` 对 GET/DELETE 无意义。只需在自己的 spider 中**覆盖**该方法即可按需支持所有 HTTP 动词：

### 重写示例

```python
import json
from scrapy import FormRequest, Request
from scrapy_redis.spiders import RedisSpider
from scrapy_redis.utils import bytes_to_str, is_dict


class MySpider(RedisSpider):
    name = "myspider"
    redis_key = "myspider:start_urls"

    def make_request_from_data(self, data):
        formatted_data = bytes_to_str(data, self.redis_encoding)

        parameter = json.loads(formatted_data)
        url = parameter.pop("url", None)
        if url is None:
            return []

        method = parameter.pop("method", "GET").upper()
        meta = parameter.pop("meta", {})

        if method in ("GET", "DELETE"):
            return Request(url, dont_filter=True, method=method, meta=meta)
        elif method in ("POST", "PUT", "PATCH"):
            return FormRequest(
                url, dont_filter=True, method=method,
                formdata=parameter, meta=meta
            )
        else:
            return Request(url, dont_filter=True, method=method, meta=meta)
```

### Redis 中推入的 JSON 示例

```bash
# GET
redis-cli lpush myspider:start_urls '{"url": "https://api.example.com/resource/1", "method": "GET", "meta": {"id": 1}}'

# PUT
redis-cli lpush myspider:start_urls '{"url": "https://api.example.com/resource/1", "method": "PUT", "name": "new_name", "meta": {"id": 1}}'

# DELETE
redis-cli lpush myspider:start_urls '{"url": "https://api.example.com/resource/1", "method": "DELETE", "meta": {"id": 1}}'
```

### 原理

| 方法 | 使用的类 | body 传参方式 |
|---|---|---|
| GET / DELETE | `scrapy.Request(method=...)` | 无 body |
| POST / PUT / PATCH | `scrapy.FormRequest(formdata=...)` | 自动编码为表单 |

`scrapy.Request` 本身支持 `method` 参数，`FormRequest` 是它的子类，额外增加了 `formdata`。按方法分派即可干净支持所有 HTTP 动词。无需修改 scrapy-redis 源码。
